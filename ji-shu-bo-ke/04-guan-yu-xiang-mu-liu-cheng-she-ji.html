工作流引擎 流程设计

工作流引擎 流程设计

表结构设计

flow_define(流程定义)

id: 主键 flow_define_code: 流程定义编码 flow_name: 流程名称 description: 流程描述 flow_define: 流程定义(json 字符串) create_time: 创建时间 update_time: 更新时间 is_deleted: 是否删除

flow_work(流程实例) id: 主键 flow_define_code: 流程定义编码 flow_name: 流程名称 last_operator: 最后一个操作者 id current_node_id: 当前节点 id next_node_id: 下一步节点 id flow_define: 流程定义（json 字符串） status: 流程状态(10->开始 START,30->结束 END,40->取消 CANCEL) flow_param: 流程参数 json create_time: 创建时间 update_time: 更新时间 is_deleted: 是否删除

flow_task(流程任务) id: 主键 flow_work_id: 流程实例 id flow_node_id: 流程节点 id task_name: 任务名称 operator: 操作者用户 id actor_user_id: 执行用户的 id status: 任务状态(10->新建 CREATED,20->已完成 FINISHED,30->过期 EXPIRED,40->取消 CANCEL,50->驳回 REJECT) service_type: 业务类型 service_id: 关联的业务 id finish_time: 完成时间 flow_param: 流程参数 json create_time: 创建时间 update_time: 更新时间 is_deleted: 是否删除

权限系统流程设计

role(角色表) id: 角色 id name: 角色名称 nameZh: 角色的中文名称。

user(用户表) id: 用户 id name: 用户名称 account: 账号名 password: 密码

user_role(用户角色表) id: 关联 id user_id: 用户 id role_id: 角色 id

menu(菜单表) id: 菜单 id url: 资源路径 path: 路由路径 component: 组件/可由前端匹配 name: 名称 icon: 图标 parentId: 父节点 id

menu_role(菜单角色关联表) id: 主键 menu_id: 菜单 id role_id: 角色 id

角色和资源的对应关系

每个菜单对应一个菜单表的实例: 菜单具有递归关系 通过角色查找菜单的 id 数组, 然后生成具有父子关系的菜单 json 返回前端
